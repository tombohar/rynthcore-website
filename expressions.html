EXPRESSIONS

Complete reference for Asheron's Call expression variables. Use bracketed syntax in meta conditions, loot filters, and chat actions.

Syntax Overview

RynthAi expressions use a familiar bracketed syntax to query game state. Variables are wrapped in square brackets [like this] and can be combined with operators and logic.

Condition Example

[SlotCount] < 5 && [Burden] > 80

Triggers when fewer than 5 slots AND burden exceeds 80%.

Chat Action Example

/g Help! My burden is [Burden]% and I only have [SlotCount] slots!

Broadcasts dynamic values to group chat.

Common Patterns & Recipes

Inventory Management

[SlotCount] < 5

Trigger when fewer than 5 free slots

[Burden] > 90 && [SlotCount] < 3

Critically full inventory

Combat Triggers

[Health] < 30

Low health - transition to healing

[Health] < 50 || [Mana] < 20

Low on either vital

[AttackerCount] > 2

Multiple enemies - switch to AOE mode

Dynamic Chat Messages

/g Need help! [AttackerCount] enemies attacking and health [Health]%

Real-time combat status broadcast

/t [FellowLeader], inventory full: [Burden]%

Report to fellowship leader

Advanced Conditions

([Health] > 75 && [Mana] > 60) && ![Encumbered]

Ready for combat - healthy, mana full, not encumbered

[TargetExists] && [TargetDistance] < 30 && ![TargetDead]

Valid combat target nearby

Tips & Notes

Case Sensitivity

Variable names must match exactly: [Health] works, [health] does not.

Whitespace is Flexible

Spaces around operators are optional: [Health]<30 and [Health] < 30 are identical.

Boolean Values

In conditions, 0 = false, any non-zero = true. Use explicitly with == 0 or != 0.

Type Coercion

The engine automatically converts between types where sensible. Mixing strings and numbers works in most cases.

Performance Tip

Simple comparisons are fast. Avoid deeply nested logic with many || branches in high-frequency checks.

Debugging

Use chat actions to print variable values: /t [Health] [Mana] [Burden] to verify in-game state.
